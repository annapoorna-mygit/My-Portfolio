Hardware Triage Tool (HTT) Internal

Automated Hardware Failure Diagnosis & Remediation System @ HPE

Project Overview

The Hardware Triage Tool (HTT) is an automated diagnostic CLI tool that helps diagnose various node hardware failures in large-scale HPC clusters and provides recommended actions for each scenario. System administrators can quickly identify failure root causes and take corrective actions, reducing manual triage time from hours to minutes.

Impact: HTT automates hardware failure detection and provides actionable repair recommendations. It supports 11+ hardware platforms, detects 6 major failure scenarios, and generates structured JSON output for downstream automation.

Failure Scenarios Detected

HTT identifies the following types of node failure scenarios:

  • Power-on Failure: A node fails to power on
  • Boot Failure: A node powers on but fails to boot (stuck at UEFI shell)
  • Health Check Failure: A node boots but fails the health check
  • Unexpected Reboot: A node reboots unexpectedly
  • Emergency Power Down: A node powers down unexpectedly (EPD)
  • Unresponsive Node: A node becomes unresponsive during job run

CLI Interface

HTT provides a comprehensive CLI with extensive options for flexible diagnostics:

  • Live Triage: Run diagnostics directly on nodes in real-time
  • Offline Analysis: Analyze existing log files for post-mortem diagnosis
  • Custom Workflows: Use custom YAML workflows for specialized diagnostics
  • Hardware Configuration: Specify hardware type and custom configs
  • Stage Control: Run specific stages or start from a particular stage
  • Serial Number Collection: Gather component serial numbers for tracking
  • Verbose Mode: Detailed output for debugging

Technical Architecture

YAML-Based Workflow Engine:

  • Customizable workflow definitions for each hardware platform and node state
  • Separate workflows for ON and OFF node conditions
  • Stage-based processing with configurable actions and branching logic
Workflow Design: Each workflow stage defines search patterns, conditions, and recommended actions. Stages can branch to other stages based on detection results, enabling complex diagnostic flows.

Hardware Configuration System:

  • YAML-based hardware baseline configurations for each platform
  • Automatic hardware mismatch detection (BIOS version, NIC count, firmware)
  • Script to generate and update hardware baselines: generate_hardware_yaml.py

Custom Scripts Framework:

  • Extensible custom script system for specialized diagnostics
  • Scripts return codes (0=detected, 1=not detected) for workflow branching
  • Support for inline arguments (node controller, log path, etc.)
